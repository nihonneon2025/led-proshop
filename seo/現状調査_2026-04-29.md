## 1. エグゼクティブサマリ

### 1-1. 一言評価

**SEO 対策度: 1.5 / 5（致命的に不足）**

Googlebot がサイトを正常にクロール・評価するために必要な要素の大半が欠落している。
トップページに `<h1>` タグが存在せず、robots.txt も sitemap.xml も 404 返却という状態は
「Googlebot がサイトの存在を認識する手がかりが何もない」に等しい。
商品詳細ページには Product 構造化データが整備されているが、それ以外はほぼ手つかず。
「誰もカタログにたどりつけない」という社長の指摘は技術的事実として正確。

### 1-2. 致命的に欠けている要素 Top 3

| 順位 | 欠如内容 | 影響 |
|---|---|---|
| 1 | **robots.txt が 404** | Googlebot がクロール設定を読めない。Search Console でエラーが発生し続ける |
| 2 | **sitemap.xml が 404** | 172 本の商品ページが Google に存在を知らせる手段がゼロ |
| 3 | **トップページに `<h1>` タグが存在しない** | Google がサイトのメインテーマを判定できない。完全な画像スクロールのみのヒーローで、テキストの一次情報が皆無 |

### 1-3. すぐ直せば改善する要素 Top 3

| 順位 | 対応内容 | 期待効果 |
|---|---|---|
| 1 | **robots.txt と sitemap.xml の新規作成・配置** | クロール効率化・商品 172 ページのインデックス登録促進。1〜2 時間の作業 |
| 2 | **トップページに `<h1>` を 1 本追加**（「LED照明部材 プロ向け総合カタログ」等） | Google のページ主題認識が明確化。数行の HTML 修正 |
| 3 | **トップページに OGP・canonical・JSON-LD（WebSite / Organization）追加** | SNS シェア時の見た目改善・正規 URL 確立・構造化データ搜索結果リッチ化 |

### 1-4. 直すのに時間がかかる要素 Top 3

### D. コンテンツ構造

#### D-1. `<h1>` タグ

- **トップページ: 存在しない**（NG・致命的）
  - ヒーローセクションは `<section class="hero hero-scroll-only">` で、内部は画像の無限スクロールのみ
  - CSS に `.hero h1` のスタイル定義はあるが、実際の `<h1>` HTML 要素がソースに存在しない
  - Googlebot がページの主テーマを判断するための最重要テキスト情報が欠落している
- **商品詳細: 1ページ1個、適切に設定済み**（OK）
  - 例: `<h1>LED COBテープライト24V ホワイト3000K</h1>`

#### D-2. `<h2>` `<h3>` の階層構造

- **トップページ**: h1 が存在しない中で h2 が複数存在する（`LEDテープライト` / `アルミフレーム` / `シリコンカバー` 等）
  - h1 の欠落により階層が崩壊している
- **商品詳細**: h1 の下に h2 等は実質なし（仕様表・説明はテーブル・段落で構成）

#### D-3. `<img alt="">` の付与率

- **ヒーロー画像 20 枚: 全て `alt=""`**（NG）
  - `aria-hidden="true"` でスクリーンリーダーには隠されているが、alt="" のまま
- **商品画像（カルーセル）**: JS で動的生成され `alt="商品画像"` という固定テキスト（部分的）
  - 具体的な製品名・色名・型番を含む alt テキストへの改善が必要
- **カテゴリ画像・製品詳細画像**: 個別の意味のある alt なし

#### D-4. パンくずリスト

- **トップページ: 未設定**（NG）
- **商品詳細: HTML のみ設定済み**（部分的）
  - `<div class="breadcrumb">` 内に `トップ > LEDカテゴリ > 商品名` の HTML 構造はある
  - ただし BreadcrumbList の JSON-LD がなく、Google に構造化パンくずとして認識されない

#### D-5. 内部リンク構造

- **トップページ → 商品詳細**: ナビゲーション内の `#cat1` 等はページ内アンカーであり、個別商品 HTML へのリンクは JS 経由（NG）

  - 静的な `<a href="products/COB24-W3K.html">` 形式のリンクが存在しない
- **カテゴリ → カテゴリのクロスリンク**: 未設定
- **関連商品リンク**: 未設定

#### D-6. アンカーテキストの妥当性

- 「こちら」「クリック」などの無意味なアンカーは確認されなかった（OK）
- カテゴリナビのリンクテキストは `LEDテープライト` / `アルミフレーム` 等、意味のある語句

- **HTTP → HTTPS リダイレクト: 設定済み**（OK）
  - `http://led-proshop.com/` → 301 → `https://www.led-proshop.com/`
- **非 www → www リダイレクト: 設定済み**（OK）
  - `https://led-proshop.com/` → 301 → `https://www.led-proshop.com/`

#### E-5. 重大：画像 URL の日本語ディレクトリ問題

- **NG（SEO 上重大）**: 画像パスが日本語フォルダ名を含む
  - 例: `https://led-proshop.com/images/テープライト/ホワイト3000K/main.png`

  - 英語名のヒーロー画像（`/images/hero-scenes/red_shop.png`）は正常に 200 を返している

## 4. 追記事項（調査中に判明した事実）

### 4-1. ホスティング環境

- **本番: GitHub Pages**（Server: GitHub.com が HTTP ヘッダーに確認）
- netlify.toml が存在するが、本番は Netlify ではなく GitHub Pages で稼働
- `https://led-proshop.com/` → `https://www.led-proshop.com/` への 301 は GitHub Pages の Custom domain 設定によるもの

### 4-2. サイト構造の重大な問題（SEO 根本課題）

- トップページ（index.html）は **8,884 行の単一 HTML** で全カテゴリ・全商品を収録
- 商品グリッドは **JavaScript で動的生成**される
- Googlebot は JS を実行するが、動的生成されたコンテンツはクロール優先度・タイミングが静的 HTML より遅れる
- 現状では Google が商品情報を取得するには products/*.html（172 ファイル）を個別クロールする必要があるが、そのためのリンク構造（静的 `<a>` タグ）がトップページに存在しない
- **sitemap.xml の作成が単なる「あると便利」ではなく「唯一の発見経路」となっている**

### 4-3. 画像 URL の日本語パス問題（実害あり）

- 本番環境で `https://www.led-proshop.com/images/テープライト/...` が 404
- OGP の og:image、JSON-LD の image フィールドに影響
- 開発ソース側の画像ディレクトリ名が日本語（`テープライト/`, `シリコンカバー/` 等）
- 英語名への変更 + generate スクリプト修正 + 既存 HTML 再生成が必要
